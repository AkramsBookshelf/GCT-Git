## Git Vs GitHub

As a beginner, you might often hear "Git" and "GitHub" used interchangeably, but it‚Äôs important to make the distinction that they are not the same.

Git is a version control application that must be installed on your computer to function. It lives locally on your machine and allows you to track changes to your project over time by creating snapshots (called commits). If something breaks, Git acts as your safety net, letting you look back at earlier versions or restore previous states of your project.

GitHub, on the other hand, is a cloud service. It stores your Git repositories online so they can be backed up, shared, and accessed remotely. Think of GitHub as the "cloud home" for the work you create using the Git application.

Before we can implement Git version control in our projects, we first need to ensure the application is installed on our local computers.
